# 4. Voice-First Architecture

## The AVID System (Patent Pending)

### What is AVID?

**AVID (Augmentalis Voice ID)** is our proprietary identification system that assigns unique, voice-friendly identifiers to every UI element.

```
┌─────────────────────────────────────────────────────────────────┐
│                    TRADITIONAL UI ELEMENT                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  <button id="btn_submit_form_12345">Submit</button>              │
│                                                                   │
│  Problems:                                                        │
│  • ID is machine-readable, not human-readable                    │
│  • No voice command association                                  │
│  • No spatial positioning data                                   │
│  • No accessibility metadata                                     │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    AVANUES UI ELEMENT WITH AVID                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  AVID: {                                                         │
│    id: "submit-button",                                          │
│    voiceLabel: "Submit",                                         │
│    voiceAliases: ["send", "done", "finish", "confirm"],          │
│    spatialPosition: { row: 3, column: 2 },                       │
│    neighbors: { up: "password-field", down: "forgot-link" },     │
│    a11y: { role: "button", hint: "Submits the login form" },     │
│    screen: "login-screen",                                       │
│    parent: "form-container"                                      │
│  }                                                               │
│                                                                   │
│  Enables:                                                        │
│  • "Click submit" → targets this element                        │
│  • "Go down" → moves to forgot-link                             │
│  • "What can I do?" → lists available actions                   │
│  • Full accessibility without extra code                        │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### AVID Capabilities

| Capability | How It Works | User Experience |
|------------|--------------|-----------------|
| **Direct Targeting** | AVID matches voice input to element | "Click the save button" → saves |
| **Spatial Navigation** | AVID tracks neighbor relationships | "Go up" → moves to element above |
| **Contextual Reference** | AVID understands conversation | "Change that color" → knows which |
| **Discovery** | AVID exposes available actions | "What can I do here?" → lists options |
| **Hierarchy Awareness** | AVID tracks parent/child | "Go back" → navigates to parent |

# 5.7 Deep Dive: RAG/LLM Integration

## Retrieval-Augmented Generation in Avanues

### What is RAG?

**RAG (Retrieval-Augmented Generation)** combines document search with AI generation. Instead of the AI making things up, it:

1. **Retrieves** relevant documents from your knowledge base
2. **Augments** the LLM prompt with that context
3. **Generates** accurate, grounded responses

### Advanced RAG Patterns

#### Pattern 1: Multi-Source RAG

Search multiple knowledge bases and combine results:

```yaml
WFL:comprehensive_search:
  TRIGGER:voice:"search everything,find all,comprehensive"
  ACTION:store_context:query

  # Search multiple sources in parallel
  PARALLEL:
    ACTION:rag_search:products:{query}:top_k=3
    ACTION:rag_search:support_docs:{query}:top_k=3
    ACTION:rag_search:internal_wiki:{query}:top_k=3

  # Combine and deduplicate
  ACTION:rag_combine:all_results:dedupe=true:rerank=true

  # Generate response from combined context
  ACTION:llm:assistant:synthesize:{query}:{combined_results}
  ACTION:speak:{llm_result.answer}
  ACTION:show:SearchResults:combined_results:with_sources
```

#### Pattern 2: Conversational RAG with Memory

Maintain conversation context across turns:

```yaml
LLM:conversational_agent:
  MODEL:claude-3-opus
  MEMORY:conversation_history:turns=10
  CONTEXT:user_profile,current_screen,recent_actions

WFL:chat:
  TRIGGER:voice:"*"  # Catch-all for conversation

  # Check if follow-up question
  ACTION:llm:classifier:is_followup:{input}:{conversation_history}

  IF:is_followup:
    # Use previous context
    ACTION:llm:conversational_agent:answer:{input}:{previous_context}
  ELSE:
    # Fresh RAG search
    ACTION:rag_search:knowledge_base:{input}
    ACTION:llm:conversational_agent:answer:{input}:{rag_results}

  # Update conversation memory
  ACTION:memory:add:{input}:{response}
```

#### Pattern 3: Agentic RAG with Tool Use

LLM decides which tools to use:

```yaml
LLM:agent:
  MODEL:claude-3-opus
  TOOLS:
    - rag_search: Search knowledge bases
    - api_call: Call external APIs
    - calculate: Perform calculations
    - navigate: Navigate to screens

WFL:agent_chat:
  TRIGGER:voice:"help me,I need to,can you"
  ACTION:store_context:request

  # Let LLM decide what to do
  ACTION:llm:agent:plan_and_execute:{request}

  # LLM might respond with:
  # 1. Tool call: rag_search("shipping policy")
  # 2. Tool call: api_call("GET /orders/123")
  # 3. Final response based on tool results

  ACTION:speak:{agent_result.response}
  ACTION:execute:agent_result.actions
```

#### Pattern 4: RAG with Structured Extraction

Extract structured data from documents:

```yaml
WFL:extract_contract_terms:
  TRIGGER:voice:"extract terms,pull out,get the details"
  ACTION:ask:Which contract?
  STORE:contract_ref

  # Retrieve the contract
  ACTION:rag_retrieve:contracts:{contract_ref}

  # Extract structured data
  ACTION:llm:extractor:extract_terms:{retrieved_doc}:schema={
    parties: string[],
    effective_date: date,
    term_length: string,
    value: currency,
    key_obligations: string[],
    termination_clauses: string[],
    renewal_terms: string
  }

  ACTION:speak:Contract between {llm_result.parties}.
         Value: {llm_result.value}.
         Term: {llm_result.term_length} starting {llm_result.effective_date}.
  ACTION:show:ContractSummary:llm_result
```

# 9. Developer Experience

## Two Paths: Zero-Code or Low-Code

### Path 1: Zero-Code (Non-Developers)

Use pre-built templates and modules. No programming knowledge required.

**Step 1: Choose a Template**
```bash
avanues create my-app --template conference-app
```

**Step 2: Customize in Visual Editor**
- Open Theme Builder
- Change colors, fonts, logo
- No code touched

**Step 3: Build & Ship**
```bash
avanues build --all
# Done. Apps for all platforms.
```

### Path 2: Low-Code (Developers & Power Users)

Write simple DSL definitions for custom UIs.

**Step 1: Install Avanues CLI**

```bash
# macOS/Linux
brew install avanues-cli

# Windows
winget install Augmentalis.AvanuesCLI

# Or via npm
npm install -g @avanues/cli
```

**Step 2: Create New Project**

```bash
avanues create my-voice-app
cd my-voice-app
```

**Step 3: Define Your UI (Simple Text Format)**

```yaml
# screens/home.amf
---
schema: amf-lyt-1.0
---
LYT:HomeScreen:1.0.0
COL:root:1.0:center
  TXT:greeting:Hello, Voice!:h1
  BTN:action:Say Something:speakAction:primary
END:root
```

**Step 4: Add Pre-Built Modules (One Line Each)**

```yaml
# app.avu - Add powerful features with one line
USE:AvaConnect:video-conference    # Full video calling
USE:VoiceControl:full-interface    # Complete voice control
USE:SecureAuth:biometric           # Secure authentication
USE:OfflineFirst:enabled           # Offline-first architecture
USE:DataSync:real-time             # Real-time synchronization
```

**Step 5: Generate & Run**

```bash
# Generate for all platforms
avanues build

# Run on Android
avanues run android

# Run on iOS
avanues run ios

# Run on Web
avanues run web
```

**What You Get:**
- Native Android app (Kotlin + Compose)
- Native iOS app (Swift + SwiftUI)
- Web app (React + Material-UI)
- Desktop app (Compose Desktop or Tauri)
- All with voice control, video calling, authentication—without writing any platform-specific code

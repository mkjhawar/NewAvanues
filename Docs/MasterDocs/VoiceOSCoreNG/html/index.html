Voice Command Processing Engine

VoiceOSCoreNG is a Kotlin Multiplatform (KMP) library that powers voice-controlled accessibility for Android, iOS, and Desktop platforms. It provides a unified architecture for speech recognition, command interpretation, and action execution with a 5-level priority-based matching system.

🔍 Smart Command Matching

5-level priority matching: Dynamic VUID, Fuzzy match, Static handlers, NLU classification, LLM interpretation

🔌 KMP Architecture

Share code across Android, iOS, and Desktop with platform-specific executors for native performance

🎤 Multiple Speech Engines

Support for Android STT, Google Cloud, Azure, Vivoka, Vosk, and Apple Speech with unified interface

🛠 Handler System

Priority-based handlers for System, Navigation, App, UI, Device, Input, and custom actions

Handler Priority Reference

Priority Category Handler Example Actions
1 SYSTEM SystemHandler back, home, recents, notifications, lock screen
2 NAVIGATION NavigationHandler scroll up/down/left/right, swipe, page up/down
3 APP AppHandler open maps, launch youtube, start camera
4 GAZE GazeHandler Eye tracking actions (future)
5 GESTURE GestureHandler pinch, zoom, drag
6 UI UIHandler click, tap, long press, toggle, focus
7 DEVICE DeviceHandler volume up/down, brightness, flashlight
8 INPUT InputHandler type, delete, paste, select all, undo
9 MEDIA MediaHandler play, pause, next, previous
10 ACCESSIBILITY A11yHandler speak, describe
11 CUSTOM CustomHandler User-defined extensions

Command Execution Priority

Step Matching Type Description Speed
1 Dynamic VUID Lookup Direct element reference via VUID from screen scraping Fastest
2 Dynamic Fuzzy Match Handles voice variations with 0.7+ confidence threshold Fast
3 Static Handler System commands registered in HandlerRegistry Fast
4 NLU Classification Semantic matching via BERT-based model Medium
5 LLM Interpretation Natural language fallback using LLM Slow
6 Voice Interpreter Legacy keyword mapping (fallback) Medium

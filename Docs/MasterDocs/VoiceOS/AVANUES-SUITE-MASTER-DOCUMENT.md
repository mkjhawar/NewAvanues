2. **AVA understands** this is a money transfer, identifies "Mom" as a contact, recognizes "$500" as the amount
3. **VoiceOS scans** the banking app's screen, finds the transfer button, amount field, and contact picker
4. **AVA maps** your intent to the specific UI elements: "Transfer" button â†’ amount field â†’ contact selection
5. **VoiceOS executes** the actions: clicks Transfer, types 500, selects Mom from contacts
6. **AVA confirms**: "Transferring $500 to Mom. Should I proceed?"

**Total time: Under 2 seconds.**

## AVA's Core Capabilities

### 1. Contextual Understanding

AVA doesn't just hear wordsâ€”it understands context:

**Screen Context**: AVA knows what app you're in, what's on screen, and what actions are possible.

```
User: "Send this"
AVA thinks: "User is in email app, draft is open, 'Send' button is visible"
AVA does: Clicks the Send button
```

**Conversation Context**: AVA remembers what you've discussed.

```
User: "Find Italian restaurants"
AVA: Shows list of Italian restaurants
User: "Call the second one"
AVA: Calls the second restaurant (remembers the list)
```

**Personal Context**: AVA learns your patterns.

```
User says "my usual" every morning
AVA learns: "my usual" = open email, check calendar, play news podcast
```

### 2. Intent Classification

AVA uses a fine-tuned BERT model (MobileBERT for speed, mALBERT for multilingual) to classify user intent:

**Supported Intent Categories**:

| Category | Example Commands | Actions |
|----------|-----------------|---------|
| **Navigation** | "go back", "scroll down", "next page" | Screen navigation |
| **App Control** | "open Chrome", "close this app" | App launching/management |
| **UI Interaction** | "click Submit", "check the box" | Element interaction |
| **Text Input** | "type hello world", "delete that" | Text manipulation |
| **System** | "take screenshot", "turn on WiFi" | Device control |
| **Query** | "search for recipes", "what's the weather" | Information retrieval |
| **Media** | "play music", "pause video" | Media control |
| **Communication** | "call Mom", "send text to John" | Calls, messages |

### 3. Entity Extraction

AVA identifies and extracts key information from natural language:

| Entity Type | Example | Extracted Value |
|-------------|---------|-----------------|
| **App Name** | "open Instagram" | `instagram` |
| **Contact** | "call Mom" | Contact: Mom |
| **Amount** | "transfer $500" | `500 USD` |
| **URL** | "go to google.com" | `https://google.com` |
| **Date/Time** | "remind me tomorrow at 3" | `2026-01-17 15:00` |
| **Location** | "navigate to Central Park" | `Central Park, NYC` |

### 4. Disambiguation

When multiple elements match, AVA helps you choose:

```
User: "Click Submit"
Screen has: [Submit Order] [Submit Review] [Submit Feedback]

AVA: "I found 3 Submit buttons. Say 'one' for Submit Order,
      'two' for Submit Review, or 'three' for Submit Feedback."

User: "Two"
AVA: Clicks [Submit Review]
```

AVA learns from your usage:

**Command Learning**:

- Adapts synonym recognition to your vocabulary

**Voice Adaptation**:
- Improves recognition accuracy for your voice
- Learns your accent and speaking patterns
- Handles background noise patterns in your environment

**Preference Learning**:
- Remembers your preferred apps for actions
- Learns your typical workflows
- Adapts confirmation preferences (more/less verification)

## Speech Recognition Engines - Comprehensive Guide

Avanues Suite supports multiple speech recognition engines, allowing you to choose the right balance of **accuracy**, **language coverage**, **privacy**, and **connectivity** for your use case.

### Online vs Offline: When to Use Each

| Mode | Best For | Trade-offs |
|------|----------|------------|
| **Online (Cloud)** | Always-connected devices (phones, tablets), maximum accuracy, 125+ languages | Requires internet, data sent to cloud |
| **Offline (On-Device)** | Field service, medical, privacy-sensitive, no connectivity areas | Smaller model = fewer languages, larger models = more storage |
| **Hybrid** | Best of both - offline primary with cloud fallback | Recommended for most users |

### Use Case Recommendations

| Scenario | Recommended Engine | Mode | Why |
|----------|-------------------|------|-----|
| **Consumer (Daily Use)** | Google STT / Apple Speech | Hybrid | Best accuracy, seamless fallback |
| **Enterprise (Office)** | Azure / Google Cloud | Online | Enterprise SLA, compliance features |
| **Field Service (No WiFi)** | Vosk / Whisper.cpp | Offline | Works in remote locations |
| **Healthcare (Privacy)** | Vosk / Vivoka | Offline | HIPAA-compliant, no cloud transmission |
| **Multilingual Teams** | Whisper / Google Cloud | Online | 99-125 languages |
| **Custom Vocabulary** | Vivoka | Offline | Grammar-based, domain-specific |
| **Smart Glasses/Wearables** | Whisper.cpp (tiny) | Offline | Low latency, small footprint |

@Composable
fun VoiceEnabledForm() {
    var name by remember { mutableStateOf("") }
    var email by remember { mutableStateOf("") }

    val avanues = LocalAvanuesSDK.current

    LaunchedEffect(Unit) {
        // Register form-specific commands
        avanues.registerCommand("fill name") {
            // Focus name field, ready for dictation
            focusManager.focus(nameField)
            avanues.startDictation { text -> name = text }
        }

        avanues.registerCommand("fill email") {
            focusManager.focus(emailField)
            avanues.startDictation { text -> email = text }
        }

        avanues.registerCommand("submit form") {
            if (validateForm()) submitForm()
        }
    }

    Column {
        TextField(
            value = name,
            onValueChange = { name = it },
            label = { Text("Name") },
            modifier = Modifier.semantics {
                contentDescription = "Name input field"
            }
        )

        TextField(
            value = email,
            onValueChange = { email = it },
            label = { Text("Email") },
            modifier = Modifier.semantics {
                contentDescription = "Email input field"
            }
        )

        Button(onClick = { submitForm() }) {
            Text("Submit")
        }
    }
}

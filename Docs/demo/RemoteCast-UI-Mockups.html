Dual Streaming Modes

MJPEG Mode (Glasses)

Sender                     Receiver
  |                          |
  |  [CAST] JPEG frame        |
  | -----------------------> |
  |                          |
  |  [CMD\0] AVU command      |
  | <----------------------- |
  |                          |
  |  [IMU\0] head tracking    |
  | <----------------------- |
  |                          |

Protocol: Custom TCP (port 54321)
Codec: JPEG (per-frame compression)
Latency: ~40-80ms
FPS: 10-20 (quality dependent)
Receiver: GlassClient APK required

WebRTC Mode (Browser)

Sender                     Receiver
  |                          |
  |  WSS: SDP offer          |
  | -----------------------> |
  |  WSS: SDP answer         |
  | <----------------------- |
  |  WSS: ICE candidates     |
  | <-------- x  ----------> |
  |                          |
  |  WebRTC: VP8/H264 video  |
  | ========================>|
  |                          |
  |  WSS: touch/key commands |
  | <----------------------- |

Protocol: WebRTC P2P + WebSocket signaling
Codec: VP8 or H264 (adaptive bitrate)
Latency: ~20-50ms
FPS: up to 60
Receiver: Any browser (zero install)

Auto-Selection Logic

Receiver connects to sender
  |
  +-- Is receiver our GlassClient APK?
  |     |
  |     YES --> MJPEG mode (custom TCP, AVU commands, IMU relay)
  |     |       Lower overhead, glasses-optimized
  |     |
  |     NO --> Is receiver a browser?
  |              |
  |              YES --> WebRTC mode (P2P video, WebSocket commands)
  |              |       Higher quality, zero install
  |              |
  |              NO --> BLE HUD mode (Vuzix Z100, text only)
  |                     Ultralite SDK, push text/images via BLE
